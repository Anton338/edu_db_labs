# Запити зацікавлених осіб

## Вступ

Система управління відкритими даними – інструмент для стабільного зберігання, легкої обробки і швидкого доступу до інформації. Цей документ представляє загальний огляд системи, ключові бізнес-процеси для користувачів та вимоги до її функціоналу, стабільності та ефективності.

### Мета

Встановити критерії для різних елементів майбутнього веб-додатку, що послужать основою для планування його розробки. Охарактеризувати функціонал веб-додатку для потенційних користувачів.

Користувач натискає кнопку "Пошук" <br/> 5. Система перевіряє базу даних за вказаними параметрами (можлива SearchByFilterError) <br/> 6. Система відображає результати пошуку користувачу <br/> 7. Користувач може переглянути деталі знайдених результатів <br/> 8. Користувач закінчує взаємодію |

| **ID:**                | DownloadData                                                                                                                                                                                                                                                                                                                                                                                    |
| ---------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **НАЗВА:**             | Завантаження даних у вибраному форматі                                                                                                                                                                                                                                                                                                                                                          |
| **УЧАСНИКИ:**          | Користувач, Система                                                                                                                                                                                                                                                                                                                                                                             |
| **ПЕРЕДУМОВИ:**        | Користувач здійснив пошук і отримав результати                                                                                                                                                                                                                                                                                                                                                   |
| **РЕЗУЛЬТАТ:**         | Дані завантажені у вибраному користувачем форматі                                                                                                                                                                                                                                                                                                                                              |
| **ВИКЛЮЧНІ СИТУАЦІЇ:** | Помилка під час генерації файлу - DownloadDataError                                                                                                                                                                                                                                                                                                                                            |
| **ОСНОВНИЙ СЦЕНАРІЙ:** | 1. Користувач вивчає отримані результати пошуку <br/> 2. Користувач обирає опцію "Завантажити результати" <br/> 3. Система пропонує вибрати формат для завантаження <br/> 4. Користувач вибирає потрібний формат і підтверджує вибір <br/> 5. Система генерує файл у вибраному форматі (можлива DownloadDataError) <br/> 6. Файл автоматично завантажується на пристрій користувача <br/> 7. Користувач отримує підтвердження про успішне завантаження <br/> 8. Користувач закінчує взаємодію |

| **ID:**                | CompareData                                                                                                                                                                                                                                                                                                                                                                                                      |
| ---------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| **УЧАСНИКИ:**          | Користувач, Система                                                                                                                                                                                                                                                                                                                                                                                             |

| **РЕЗУЛЬТАТ:**         | Відображене порівняння двох наборів даних                                                                                                                                                                                                                                                                                                                                                                       |
| **ВИКЛЮЧНІ СИТУАЦІЇ:** | Набори даних не є сумісними - CompareDateError                                                                                                                                                                                                                                                                                                                                                                |
| **ОСНОВНИЙ СЦЕНАРІЙ:** | 1. Користувач розпочинає взаємодію <br/> 2. Користувач вибирає опцію "Порівняти набори даних" <br/> 3. Система пропонує користувачеві завантажити або вибрати перший набір даних для порівняння <br/> 4. Користувач завантажує або вибирає перший набір даних <br/> 5. Система пропонує користувачеві завантажити або вибрати другий набір даних для порівняння <br/> 6. Користувач завантажує або вибирає другий набір даних <br/> 7. Система аналізує обидва набори даних та здійснює порівняння (можлива CompareDateError) <br/> 8. Система відображає результати порівняння у зрозумілому для користувача форматі (наприклад, в таблиці з позначенням різниці) <br/> 9. Користувач може вивчати результати порівняння, використовуючи додаткові функції, такі як збільшення, фільтрація або сортування <br/> 10. Користувач закінчує взаємодію |
